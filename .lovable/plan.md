# Premium Product Architecture v2 — Implementation Plan

This is a large, single-pass upgrade of the existing Prompt Product Factory. No rebuild — we extend the existing engine, API wrapper, and pages. Manual-upload-only stays the default and only mode.

## 1. Scope summary

Convert the current single-bundle output into a **3-layer product system**:

- **Buyer Package** — only buyer-facing files + a real premium `Product_Handbook.pdf`.
- **Seller Master Toolkit** — one consolidated `00_Seller_Master_Toolkit.md` containing pricing, thumbnail/cover briefs, video script, per-marketplace listing drafts.
- **Admin Manifest** — `12_Product_Manifest.json` + `19_Marketplace_Bundle_Index.md`.

Plus: 3 ZIP export modes, QC v2 with stricter rules, premium content quality bar, and 7 marketplaces (Shopee, Tokopedia, Lynk.id, Gumroad, Etsy, Envato, LemonSqueezy).

## 2. Files to modify / add

- `src/lib/runner/types.ts` — extend `MARKETPLACES`, add `FINAL_BUYER_MODULES`, `SELLER_TOOLKIT_MODULE`, `ADMIN_MODULES`, `IGNORED_LEGACY_MODULES`, `PREMIUM_QUALITY_MODES`, `EXPORT_MODES`, `ProductManifestPayload v2` type, QC check IDs.
- `src/lib/runner/engine.ts` — new/updated generators: `productBrief`, `promptBook` (premium quality), `promptLibraryCsv`, `usageGuide`, `sampleInputOutput`, `buyerFAQ`, `completePdfProductDraft`, `qcScorecardTemplate`, `sellerMasterToolkit`, `productManifestJson` (v2 schema), `marketplaceBundleIndex`, `marketplaceAdapterCopy(platform, seller, adapter)`, `visualAssetPromptKit`. QC v2 rules. Forbidden-claims scanner expanded.
- `src/lib/runner/api.ts` — `buildManifest` produces only v2 modules; `generateAllRemainingFilesWithProgress` generates new module set; `runAndPersistQC` validates v2; add `buildBuyerExport`, `buildSellerExport`, `buildFullSystemExport` returning file maps; backward-compat for legacy runs (don't crash if old modules exist — just hide them).
- `src/pages/NewRun.tsx` — add Premium v2 toggle (default ON for new runs), export target chooser, marketplace checkboxes for the 7 platforms, product quality mode (Starter/Premium/Super Premium), helper text on manual-upload-only.
- `src/pages/RunDetail.tsx` — new tab set: Overview, Architecture, Buyer Package, PDF Handbook, Seller Toolkit, Marketplace Adapters, QC, Export. PDF Handbook tab renders preview of `20_Complete_PDF_Product_Draft.md` and exposes Generate Preview + Download PDF. Export tab: 3 ZIP buttons + QC summary, buyer-isolation status, manual-upload status, PDF readiness.
- `src/components/runner/ProductHandbookPreview.tsx` (new) — styled markdown preview component used for both screen preview and PDF rendering source.
- `src/lib/runner/pdf.ts` (new) — `generateProductHandbookPdf(markdown, meta)` using `jspdf` (cover, page breaks per H1, footer page numbers, clean typography). Returns `Blob`.
- `src/lib/runner/zipExport.ts` (new) — uses `jszip` to assemble the 3 ZIP modes from file maps; injects PDF into Buyer ZIP and Full ZIP; never injects seller-only files into Buyer ZIP.

## 3. Dependencies

Add (if not present): `jspdf`, `jszip`, `marked` (lightweight md→tokens for PDF section detection). All client-side. No new secrets.

## 4. Content quality rules

Every prompt in `02_PromptBook.md` gets: context, role, input variables, task steps, output format, safety rules, manual verification notes, beginner example, advanced example. Prompts seeded from `niche + audience + adapter` to avoid generic copy; uniqueness check enforced by QC.

Each marketplace listing in the Seller Toolkit is generated by `marketplaceAdapterCopy` with platform-specific tone (Shopee SEO-heavy ID, Tokopedia formal ID, Lynk.id creator-funnel, Gumroad storytelling USD, Etsy template tags, Envato structured docs, LemonSqueezy SaaS). QC fails if any two platforms have identical descriptions.

## 5. PDF handbook

Source: `20_Complete_PDF_Product_Draft.md`. Sections: Cover, Product Promise, Problem Framing, System Overview, How It Works, Buyer Workflow, Core Prompt System, Prompt Usage Examples, Beginner Workflow, Advanced Workflow, Sample Transformation, QC Checklist, FAQ, License & Safe Use. Each H1 → new page; cover page styled; footer page numbers; no seller content; not a 3–5 page stub.

## 6. QC v2

Implements all 25 checks listed in the spec. Scoring:
- `< 85` → `NOT_SELL_READY`
- `85–94` → `SELL_READY_STARTER_BETA`
- `≥ 95` → `SELL_READY_PREMIUM_DRAFT`

`PASS_FINAL` requires `≥ 95`, no blocking errors, PDF draft contains all required sections, Buyer Package contains zero seller-only filenames, manifest excludes all `IGNORED_LEGACY_MODULES`, no `API_*` modules anywhere.

## 7. Backward compatibility

- Old runs in DB keep loading. `RunDetail` detects v1 vs v2 manifest and renders the matching tab set (v1 falls back to current behavior).
- "Lemon Squeezy" string normalizes to `LemonSqueezy` on read.
- Ignored legacy files are stripped from new manifests but not deleted from existing rows.

## 8. Safety

Forbidden-claim scanner expanded: guaranteed income/sales, marketplace approval, official partner, fake testimonials/scarcity/reviews/buyer numbers, automatic marketplace publishing, financial/legal certainty. Negation-aware (already in place) so checklists can still mention them as things to avoid.

## 9. Out of scope (explicit)

- No marketplace API publishing, no auto-upload, no `API_*` modules.
- No real image generation — only image briefs/prompts in Seller Toolkit.
- No fake testimonials, sales proof, ratings, or partner claims.
- No server-side secrets.

## 10. Verification

- TypeScript build via the harness.
- Manual smoke: create a run with the spec's test inputs (RESEARCH adapter, 10 prompts, all 7 marketplaces), confirm Buyer ZIP excludes seller files, Full ZIP contains all 3 folders, PDF downloads with real sections, QC ≥ 95.

## 11. Known limitations (will be reported)

- PDF styling is clean printable (jspdf) — not a full design-studio layout. Cover, headings, page breaks, footer numbers, consistent typography. If user wants InDesign-grade visuals later, we'd swap to `@react-pdf/renderer` or a server render.
- ZIP assembly is client-side; very large runs may take a few seconds in-browser.
- Existing v1 runs keep their old tab layout; only new v2 runs get the new tabs.
